Prologue

Editing files

If you don't know anything about HTML and web pages, before doing anything with this template I recommend you to take at least some basic tutorial or hire somebody experienced to do this.

As for tutorials I recommend w3schools.com

Editing php and JavaScript files is really advanced stuff, only experienced person should edit them. If you're not sure, do not change anything, it might cause serious damage to the scripts. Hiring somebody, who knows what to do, is better solution.

If you are not familiar with these technologies, do not play around with these files.

You may use defined margin classes: "margin-10", "margin-20" ... "margin-100" ("margin-10" = element with 10px height).

Responsive layout

Responsive layout makes the content fit the window width. If content changes the responvity might get worse or even broken. This cannot be handled be template - if original menu has 6 items, the responsivity counts with this approximately width, but if this menu is made with 15 items, the original responsivity rules cannot handle such difference. So after editing content you should check, if the responsivity still works well and if not, then make some adjustments.

These responsivity rules are called media queries and usually are at the bottom of the stylesheets.

7. Contact and subscribe forms

Please note: these scripts need php support - so if you will try this on your personal computer it will probably not work (unless you have Apache server or something like that running). You have to try it on some web server and this server must have php support. PHP (and mail function) si reguired for sending emails. Most of the web servers have this support, it's basic equipment, but some might not.

Prolog

Make changes based on this documentation, do not change anything else if you are not sure, what to do, you can cause serious damage to the scripts.

Do not change contact forms, unless you know for sure what to do. Elements in contact forms must correspond to the ajax variables.

After changes are made, remember to check if it works properly.

Where to start?

Contact: open file send_email.php read the comments on the top lines and make the changes. That's all for contact form.

RSS: File save_rss.php is for RSS subscribe, the default setting is MailChimp, other options are: "save to email", "save to file".

When you pick one of these options, it needs to be activated, if you want to use mailchimp, you're done here, in other cases go to folder save_email_options pick the file you want and replace the original save_rss.php file with this file, keep the name save_rss.php

Open file save_rss.php read the comments on the top lines and make the changes. That's all for RSS form.

Mailchimp

Online email marketing solution to manage contacts, send emails and track results.

Mailchimp api docs - http://apidocs.mailchimp.com/api/

PHP files

Show content

class.emailSender.php (include/class.emailSender.php)

PHP class for sending emails.

Sending emails: this is just class file, include it to other scripts and use it there

Editing user messages: at the top of that file you can see $user_messages and $control_messages variables. They are set for two languages - EN and CZ - here you can change the messages (only the right side, do not delete whole line, it will cause errors and do not leave the message empty - for ajax empty message means everything's ok). For changing the language find variable "$language" (orignally "$language = 'EN';") and change it to another language code that is defined.

Adding new languages: copy one of the languages made, change the language code ('EN' to 'FR' for example) and change the messages (only the right side, do not delete whole line, it will cause errors and do not leave the message empty - for ajax empty message means everything's ok). Then change the language to the new one - find variable "$language" (orignally "$language = 'EN';") and change it to another language code that is defined.

How To...

Set contact and subscribe form

Please refer to section "7. Contact and subscribe forms" and subsection "Where to start?"

Change head slider

This is plugin masterslider, please refer to documentation/masterslider-documentation

Add scroll-to event

Make an element with "scroll-to" class and attribute "href" leading to the anchor like href="#top"

Add on-scroll fade effect

Add onscroll-animate class to choosen element, be carefull if this element already has animation, or element that has both animation and transition might behave weird in some browsers.

Change google map position

In file custom.js look for code "google.maps.LatLng(40.714728,-73.998672)" where 40.714728 and -73.998672 are the position coordinates.

Block emails or IPs from sending spam messages

In selected php file adjust emailSender object to something like this: "new emailSender('blacklist_ips.txt', 'blacklist_emails.txt');" where "blacklist_ips.txt" is text file with ips to block, one ip on one line. Same for blocked emails.

Note: this is implemented only for emailSender() class, so it will work only if sending to email is used (not working for Mailchimp either saving to file).

Change form messages

See form attributes like data-success-msg etc., to change php messages see section "Contact and subscribe forms" > "PHP files" > "class.emailSender.php"

FAQ

WordPress error while installing the theme

Well, this is correct, because this is not a WordPress theme. If you want to buy a WordPress theme, make sure it is in the right category - WordPress, HTML, Marketing, CMS, eCommerce...

So unfortunately I can't help you with this. And as I know Themeforest gives no refunds for templates bought by mistake. But you can still use this template, but not on WordPress.

"Ajax could not send the request" error message

This means, that php script is not available - wrong path to the script, or php is not running - if you do not have php running on your personal computer, it will not work, try it on web server and make sure the paths are correct.

Video player is not working

Note that this works only on web server, if you try it on your personal pc, it will not work.

"Problem on server side, please try again later." error message

This means, that php works, but mail function does not.

How to deal with it:

  • Upgrade your hosting, so you will be able to send email from your web server
  • Change the script - for saving emails (rss for example) three scripts are available, "mailchimp" and "save to file" do not require mail function. But sending messages from contact form requires mail function.
  • Use other mail servers - for example servers from google. This option is beyond this template and is not implemented, I recommend using PHPMailer (you can see basic example at the bottom of send_email.php). This is advanced stuff that require php knowledge.
  • If it worked before, it might be just temporary problem on server
